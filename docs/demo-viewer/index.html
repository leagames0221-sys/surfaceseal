surfaceseal
毒入りの設定変更を、

AI コーディングエージェントが自動で信頼する設定ファイル。
その変更だけを検査し、毒が混ざったプルリクエストを出口で堰き止めます。

diff-aware security merge gate · deterministic · local-first · runtime dependencies = 0

THE PROBLEMAI 助手は設定ファイルを黙って信じて動く

リポジトリを開くと、AI 助手は中の指示ファイルや仕掛けを自動で読み、人間が見る前に実行します。

  • そこに悪い指示を仕込んだプルリクエストが、実際の攻撃手口になりました
  • 感染した開発者が、気づかぬまま自分のプロジェクトにも毒を書き込んで配布します
  • こうして人から人へ自動で広がり、多数のリポジトリが停止に追い込まれました

既存の検査機は「取り込む側」を守りますが、「毒を出荷してしまう側」は空白でした。

APPROACH変更そのものだけを見る誤検知を抑える

全部を毎回調べると、もとからある正常な設定まで警告だらけになります。surfaceseal は土台と比べ、このプルリクエストが新しく持ち込んだ部分だけを判定します。

$ surfaceseal scan --diff --base origin/main
# もとからある正常なフックは素通り、今回増えた変更だけを検査

① AUTORUN自動で走る仕掛けの追加を捕まえる不合格

起動時に自動実行されるフックや、外部とつながる定義が新しく加わったら不合格。中身を一度も実行せずに、構造から見抜きます。

$ surfaceseal scan --diff --base origin/main
CRITICAL .claude/settings.json:14 [SS-CRED-PATH]
  auto-executing command added in hook:SessionStart
$ echo $? # 終了コード = 2
2

② DRIFT信頼した後のこっそり改ざんを見張る

承認した時点の姿を指紋として記録。あとから黙って書き換わっていないかを監視します。今回の自己増殖型の広がりに直接効く層です。

$ surfaceseal init # 現状を信頼済みとして記録
accepted commands; pinned surfaces
$ surfaceseal scan --baseline
  [CRITICAL] 承認後に変わった仕掛けを検出

③ INJECTION指示文に紛れた隠し命令参考・補助

指示書きのファイルに紛れ込んだ、秘密を送れ・人に隠せ、といった隠し命令の気配を拾います。文章の検出は完全ではないため、あくまで補助（警告止まり）として正直に扱います。

隠し命令の気配

秘密の外部送信や秘匿の指示 → SS-INJ（参考）

正直な位置づけ

確実な構造検査を主役に、文章検査は警告どまりで誤誘導を防止

FAIL CLOSED読めない設定は安全側に倒す

検査を妨害する壊れた設定や、許可リストにない新しい仕掛けは、黙って通さず不合格にします。

状況surfaceseal の応答
解析できない設定で検査機を惑わすSS-PARSE 読めない = 不合格
同じ命令を危険な起動条件に付け替えるSS-AUTORUN-NEW 付け替えも変更扱い
許可リストにない仕掛けを足す既定で不合格 人の承認が必要

公開済みのすり抜けを再現テストとして恒久化回帰テストで固定

CI GATEギットハブ上でマージを止める

毒入りの変更を含むプルリクエストは、終了コード 2 でチェックが失敗し、マージ自体を堰き止めます。結果は標準形式でも出力し、セキュリティ画面に並びます。

$ surfaceseal scan --diff --format sarif
# 標準形式（サリフ）で出力 → ギットハブのコードスキャンに表示
  "ruleId": "SS-CRED-PATH", "level": "error"

差分の検査と合否の全判定。エーアイ不使用・ネット不要・毎回同じ結果。
